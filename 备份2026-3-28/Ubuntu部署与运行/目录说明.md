# 「协议分析实验一」— 目录说明

以下路径均以 **项目根目录** `协议分析实验一/` 为基准（将仓库克隆或拷贝到 Ubuntu 后，进入该目录操作）。

---

## 顶层结构

| 路径 | 内容 |
|------|------|
| **`Ubuntu部署与运行/`** | 本目录：Ubuntu 构建、运行、测试说明（`README.md`、`Ubuntu构建与运行指南.md`、本文件） |
| **`实验资料/`** | 实验思路、DNS 操作说明、`scripts/` 一键脚本（安装依赖、启动临时容器） |
| **`实验结果老师版的描述/`** | 老师演示输出解读、`PDF与演示对照检查清单.md` 等 |
| **`源代码/my_nslookup/`** | **模块化源码**：`main.py`、`display.py`、`dns_parser.py`、`packet_builder.py`、`packet_sender.py`、`pcap_writer.py`、`utils.py`、`requirements.txt`、`Dockerfile.multifile` |
| **`my_nslookup/`** | **单文件版** `my_nslookup.py`（逻辑与模块化版一致）、`Dockerfile.singlefile`、`requirements.txt`（可与 `源代码` 下保持同步） |
| **`目录说明.md`**（根目录） | 简版索引与 Docker 提示 |

---

## 运行入口（任选其一）

| 方式 | 命令（均在项目根目录执行） |
|------|---------------------------|
| 模块化 | `sudo python3 源代码/my_nslookup/main.py <DNS_IP> <域名> [选项]` |
| 单文件 | `sudo python3 my_nslookup/my_nslookup.py <DNS_IP> <域名> [选项]` |

常用选项：`--iface <网卡名>`（如 `eth0`、`ens33`）、`--show all`、`--output capture.pcap`。
